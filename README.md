# ネイティブSDK開発

## 概要

現在、携わっているプロジェクトです。ネイティブSDK開発メンバーのテックリードに携わっています。メンバーの生産性の向上、コードの品質向上、コミュニケーションの活性化、メンバーのSDK技術向上などをゴールに設定して仕組み作りを行っています。

主な業務は、メンバー間の課題発見、開発レビュー、新機能や技術課題への意見、ネイティブSDKの開発、クロスプラットフォームのブリッジSDK開発、エスカレーション対応、などです。

*環境*

- GitHub

## 課題１
SDKが抱える技術的負債の解消・生産性向上・メンバー技術力向上、メンバー育成。

*課題*

チーム再編によるコアメンバーの異動による技術力の低下、大規模スクラムによるSDKメンバーの分散によりSDKが持つ課題への対応が難しくなった。

*解決方法*

この課題を解決するために、"課題"による問題と定性目標の洗い出しを行いました。結果、「コミュニケーションの量や質の低下」「ナレッジの共有の仕組みの不足」「新メンバーの育成環境がない」と言った課題が見えてきました。そのため、まずSDK開発メンバーに課題意識の共有・MTGやドキュメント化の重要性を再認識する機会を作りました。ある程度の同意を得られたため、SDKメンバーMTGの定期開催・SDKメンバーの課題を整理するフローの導入しました。また、シニアエンジニアとしても経験の浅いメンバーのメンタリング・積極的なコードレビュー・設計のサポートなどを行っています。


## 課題２

自走するチームづくり。

*詳細*

シニアエンジニアとして開発しつつもチームをリードする必要があるため、全てをサポートすることは難しことが多々あります。そのため、既存のメンバーが自走するための仕組みが必要になりました。


*解決方法*

メンバーが自走するために、メンバーが同じ一体となって成長するためDaily MTGやWeekly MTGで出来るようにゴールの認識を合わせできる様な仕組みを取り入れています。新しいメンバーが入ってきた時に、既存メンバーが育成に関わりやすくなる様にオンボーディングの資料を作成・整理したり、既存の仕様をドキュメントにまとめたりしています。また、既存メンバーのレビューや設計におけるメンバーの技術力向上、チームを跨いだタスクの整理し共有できる様にすることで課題解決をできる様に工夫しています。

## 課題１
自社システム開発のニーズはあったものの、1年以上リリース開発・リリースができずにいた。

*課題*

- 開発メンバーが少ないため、リソースが十分ではない
- 開発に関するチーム間を跨いだ積極的なコミュニケーションが少ない
- MVPやPMFを検証方法が明確になっていなかった

*解決方法*

- 編集チームの業務フローの洗い出し
- MVPの決定・PMFの検証方法の提案、
- 1週間ごとにタスクの優先度とスコープの調整

## 課題２
リリース直後にサービスの負荷が高くなり、すぐにシステムがダウンしてしまう。原因が分からず元のシステムにロールバックしていた。原因は、DBのインデックスが不足していたためDBの負荷が高まっていた。

*課題*

- 会社で自社システムの開発は初めてだったため原因に思い当たる人がいなかった
- 数千万のPVがあるメディアだったため、品質の問題はクリティカル
- 運用コストが高くなっていたため、システムの導入は大きかった
- 運用体制が移行後のものになっていたため、社内の仕事が一時止まっていた

*解決方法*

- AWS Cloud Watchを使って、フロントエンド・APIロジック・DB間の問題切り分け
- DBログとAPIリクエストから、重いクエリの調査
- 対処に必要なインデックスの洗い出しと開発環境での検証

## 課題１
大規模メッセージングアプリのアプリ内キャッシュ削除を実装しました。

*課題*

- ファイル量が多い人は、処理に時間がかかることが多い
- 削除可・削除不可のファイルが同一ディレクトリ内にある

*解決方法*
- group dispatch とdispatch async を 利用して、並行に処理を実行して処理時間を短縮して削除する
- 一気に多くのファイルを削除対象にするのではなく、複数回のリリースでファイルを仕分ける

## 課題２
トークルームの画面を、1つのビジネスアカウントを複数人で運用できるように対応した。

*課題*

- 大規模メッセージングアプリのトークルームは1アカウントに対し1ユーザー前提となっている
- メッセージ機能がメインのアプリなので、トークルームの実装は複雑
- 送信できるメディアを限定する必要がある（公式スタンプ、画像のみ）

*解決方法*

シンプルに、課題を分解しつつ1つずつ対応した。

- 少しずつリファクタリングをしつつ、ビジネスアカウントIDを利用して表示名を変える
- UIを非表示にしつつ現時点で必要ない機能は削除 or コメントアウトする
